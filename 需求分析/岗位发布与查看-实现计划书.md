## 1. 现状分析与差距识别

### ✅ 已经存在的（无需改动）

| 模块 | 已实现内容 | 路径 |
|------|-----------|------|
| **数据库表** | `job_position` 表已有 | MySQL |
| **实体类** | `JobPosition.java`（含 title/direction/jd/city/salaryRange/companyName/jobType/publishTime 等） | `entity/` |
| **Mapper** | `JobPositionMapper`（MyBatis-Plus BaseMapper） | `mapper/` |
| **企业实体** | `Enterprise.java`（公司名称/行业/规模/地址/联系人） | `entity/` |
| **学生端搜索** | `GET /api/student/jobs/search?keyword=&city=` → 返回岗位列表 | `JobMatchingController` |
| **学生端详情** | `GET /api/student/jobs/{id}` → 返回岗位详情 | `JobMatchingController` |
| **学生端推荐** | `POST /api/student/jobs/recommend` → AI 匹配推荐 | `JobMatchingController` |
| **文件上传** | `FileUtil.java`（校验/保存/删除本地文件） | `util/` |
| **JWT鉴权** | `JwtInterceptor` + `RoleInterceptor` | `interceptor/` |
| **前端路由** | `/enterprise/jobs` 路由已注册 | `router/index.ts` |
| **前端岗位列表** | `JobPostView.vue`（用 `el-table` 展示岗位） | `views/enterprise/` |
| **前端API** | `searchJobs()`, `getJobDetail()`, `recommendJobs()` | `api/student.ts` |

### ❌ 缺失的（需要新增）

| 缺口 | 说明 |
|------|------|
| **HR 没有创建岗位的 API** | `EnterpriseController` 只有 recommend 方法，没有 add/update/delete |
| **前端没有发布岗位的表单** | `JobPostView.vue` 只有表格 + 空按钮，点击无反应 |
| **前端 `enterprise.ts` 缺少 API 函数** | 只有 `recommend()`，没有岗位 CRUD |
| **JobPosition 缺少关键字段** | 没有 `hrUserId`（谁发布的）、`publishStatus`（草稿/已发布状态） |
| **FileUtil 不支持图片** | 只支持 PDF/DOCX，岗位封面图需要 JPG/PNG |
| **前端学生端没有岗位浏览页** | 学生搜索API已有，但前端没有岗位列表/详情展示页 |

## 2. 实现思路

```
┌─────────────────────────────────────────────────────────────┐
│                      整体数据流                               │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  ┌─────────────────┐         ┌─────────────────┐            │
│  │   HR 登录        │         │   学生登录       │            │
│  │   role=HR        │         │   role=STUDENT   │            │
│  └────────┬────────┘         └────────┬────────┘            │
│           │                            │                      │
│           ▼                            ▼                      │
│  ┌─────────────────┐         ┌─────────────────┐            │
│  │ POST /api/      │         │ GET /api/student│            │
│  │ enterprise/jobs │  ────▶  │ /jobs/search    │            │
│  │ （创建岗位）      │  写入   │ （搜索岗位）      │  读取      │
│  └────────┬────────┘         └────────┬────────┘            │
│           │                            │                      │
│           ▼                            ▼                      │
│  ┌─────────────────────────────────────────┐                 │
│  │              MySQL job_position          │                 │
│  │  (hr_user_id, publish_status=1, ...)    │                 │
│  └─────────────────────────────────────────┘                 │
│                                                               │
│  思路：在已有读写链路上补"写入"缺口，前端补"发布表单"即可        │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

**核心策略**：
- **后端**：复用已有 `JobPositionMapper` + `JobMatchingService`，在 `EnterpriseController` 中新增 create/update/delete 三个接口
- **JobPosition 实体**：新增 2 个字段（`publishStatus`、`hrUserId`），改已有表
- **前端**：补发布表单 + 补学生端岗位浏览页 + 前端 API
- **不引入任何新中间件**：不用 ES、不用 MQ、不用 MinIO

### 4.5 JobMatchingServiceImpl 实现

```java
// ===== 在 JobMatchingServiceImpl 中新增 =====

@Override
@Transactional
public JobPosition addJob(CreateJobRequest request, Long hrUserId) {
    JobPosition job = new JobPosition();
    BeanUtils.copyProperties(request, job);
    
    job.setHrUserId(hrUserId);
    job.setPublishStatus(1);  // 1 = 已发布
    job.setPublishTime(LocalDateTime.now());
    
    jobPositionMapper.insert(job);
    log.info("岗位发布成功: id={}, title={}, hrUserId={}", 
        job.getId(), job.getTitle(), hrUserId);
    return job;
}

@Override
@Transactional
public JobPosition updateJob(Long jobId, CreateJobRequest request, Long hrUserId) {
    // 校验岗位存在且属于该HR
    JobPosition job = jobPositionMapper.selectById(jobId);
    if (job == null) {
        throw new BusinessException(404, "岗位不存在");
    }
    if (!hrUserId.equals(job.getHrUserId())) {
        throw new BusinessException(403, "无权修改此岗位");
    }
    
    BeanUtils.copyProperties(request, job);
    job.setId(jobId);  // 防止 id 被覆盖
    job.setHrUserId(hrUserId);
    
    jobPositionMapper.updateById(job);
    log.info("岗位更新成功: id={}", jobId);
    return job;
}

@Override
@Transactional
public void deleteJob(Long jobId, Long hrUserId) {
    JobPosition job = jobPositionMapper.selectById(jobId);
    if (job == null) {
        throw new BusinessException(404, "岗位不存在");
    }
    if (!hrUserId.equals(job.getHrUserId())) {
        throw new BusinessException(403, "无权删除此岗位");
    }
    
    // MyBatis-Plus 逻辑删除（@TableLogic）
    jobPositionMapper.deleteById(jobId);
    log.info("岗位已下架: id={}", jobId);
}

@Override
public List<JobPosition> listByHrUser(Long hrUserId) {
    return jobPositionMapper.selectList(
        new LambdaQueryWrapper<JobPosition>()
            .eq(JobPosition::getHrUserId, hrUserId)
            .eq(JobPosition::getIsDeleted, 0));
}
```

### 流程 1：HR 发布岗位

```
┌────────┐    ┌──────────────┐    ┌──────────────────┐    ┌───────┐
│ HR用户  │───▶│ JobPostView  │───▶│ EnterpriseController│──▶│ MySQL  │
│ 登录后  │    │ (填写表单)    │    │ POST /api/enterprise│    │ job_   │
│        │    │              │    │ /jobs              │    │position│
└────────┘    └──────────────┘    └──────────────────┘    └───┬───┘
                                                               │
                            publish_status = 1                  │
                            hr_user_id = 当前HR的ID              │
                                                               │
    ┌──────────────────────────────────────────────────────────┘
    │
    ▼
┌────────────┐    ┌──────────────────┐    ┌───────────────┐
│ 学生用户    │───▶│ JobListView      │───▶│ JobMatching   │
│ 登录后      │    │ (搜索/浏览岗位)   │    │ Controller    │
│            │    │                  │    │ GET /api/     │
└────────────┘    └──────────────────┘    │ student/jobs/│
                                          │ search        │
                                          └───────┬───────┘
                                                  │
                                                  ▼
                                          ┌───────────────┐
                                          │ WHERE publish_│
                                          │ _status = 1   │
                                          │ (只查已发布)   │
                                          └───────────────┘
```

### 关键保障

| 保障点 | 实现方式 |
|--------|----------|
| 只有 HR 能发布 | `RoleInterceptor` 拦截 `/api/enterprise/**`，校验 role=HR |
| 学生不能修改岗位 | 修改接口在 `/api/enterprise/**` 路径下，学生进不来 |
| 学生只能看已发布 | `searchJobs()` 查询加 `publishStatus = 1` 条件 |
| HR 只能操作自己的岗位 | `updateJob/deleteJob` 校验 `job.hrUserId == 当前userId` |
| 图片上传安全 | `FileUtil` 校验类型/大小，UUID 重命名防覆盖 |
