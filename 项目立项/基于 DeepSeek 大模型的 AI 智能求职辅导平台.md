# 1 项目提出

**项目名称**：基于 DeepSeek 大模型的 AI 智能求职辅导平台

**项目简介**：

2025 年全国高校毕业生人数突破 1200 万，就业市场竞争空前激烈。调查显示，超过 70% 的大学生在毕业前对自身职业发展方向感到迷茫——"不知道自己喜欢什么、适合什么、能做什么"成为困扰当代大学生的首要就业难题。尤其在互联网行业，岗位类型繁多（前端/后端/产品/数据/设计/运营/测试等）、技术栈迭代快、岗位要求差异大，许多学生盲目跟风"转码"却因不了解各岗位的真实技能要求而半途而废。与此同时，互联网企业同样面临"招人难"困境——HR 高度依赖关键词筛选简历，难以识别具备实际项目能力的候选人。

本项目基于 DeepSeek 大模型技术，构建面向大学生与互联网企业双向赋能的 AI 智能求职辅导平台。对大学生用户，平台以"发现方向 → 定位差距 → 提升能力 → 精准匹配"为核心路径：通过职业方向探索和 AI 对话式引导帮助其发现适合的互联网岗位方向，基于岗位标准化技能要求生成个人能力差距分析报告，自动规划分阶段学习路径，辅以技能学习、简历优化、模拟面试等全流程辅导，最终实现人岗精准匹配。对企业用户，平台提供超越关键词搜索的智能人才推荐能力——HR 不仅可按岗位搜索候选人，还可通过输入项目功能描述，让 AI 自动推演项目所需的岗位类型和技能组合，从人才库中智能推荐匹配候选人，实现从"项目需求"到"人才匹配"的智能化闭环。

**项目目标**：

1. 构建面向互联网行业的用户求职画像与职业能力基线测评体系，通过五维度测评精准定位用户初始能力等级与潜在优势方向
2. 基于 DeepSeek 大模型实现 AI 职业方向探索与规划咨询，以对话式引导帮助用户完成自我探索并推荐 3-5 个最适合的互联网岗位方向
3. 建立岗位智能匹配与推荐机制，利用多维度语义匹配算法实现人岗精准匹配，附带匹配度评分与技能对比雷达图
4. 实现 AI 岗位能力差距分析功能，自动解析目标岗位技能要求，生成量化的能力差距分析报告
5. 基于能力差距自动规划"基础夯实→框架掌握→项目实战→面试冲刺"四阶段个性化学习路径
6. 搭建互联网岗位技能学习模块，覆盖前端/后端/产品/数据/测试五大方向核心知识体系
7. 实现 AI 简历智能优化与 AI 模拟面试评估功能，提供从简历到面试的全流程求职辅导
8. 构建企业端智能人才推荐系统，支持按岗位搜索候选人和项目驱动人才智能推荐两种模式

**系统边界**：

系统分为学生端（Web端）、企业端（Web后台）与运营管理端（Web后台）三大入口，聚焦 MVP 核心功能：

- **学生端**：用户注册与求职画像、职业能力基线测评、AI 职业方向探索与规划咨询、岗位智能匹配与推荐、AI 岗位能力差距分析、个性化学习路径规划、互联网岗位技能学习、AI 简历智能优化、AI 模拟面试与表现评估、学习进度追踪与激励
- **企业端**：企业岗位人才搜索、项目驱动人才智能推荐、企业管理后台
- **管理端**：高校管理后台、运营管理平台
- **智能客服**：基于 DeepSeek 的 AI 智能客服系统

# 4 风险评估和规避

## 技术风险

1. **AI 岗位匹配准确性风险**：DeepSeek 大模型在进行岗位推荐和能力差距分析时可能出现偏差或"幻觉"，若 AI 推荐的方向与用户实际能力不匹配，可能误导用户的职业选择和求职方向。

2. **简历语义解析准确性风险**：用户上传的简历格式多样、内容高度个性化，AI 需要从非结构化文本中准确提取技能、项目经验、教育背景等关键信息，解析错误将导致岗位匹配失准。

3. **大模型 API 依赖风险**：核心功能（职业方向探索、AI讲解、模拟面试、智能客服、简历优化）均依赖 DeepSeek API 的可用性，存在服务中断或响应延迟风险。

4. **技术栈学习曲线风险**：3 人团队需在短时间内掌握 DeepSeek API 接入、RAG 技术、语义匹配算法、前后端全栈开发，团队成员技术水平参差不齐可能导致进度受阻。

解决：

1. 采用 RAG（检索增强生成）技术，将 AI 推荐约束在互联网行业岗位技能标准化词典和真实供需数据范围内；建立推荐结果的可解释性机制（逐项说明推荐理由）；对 AI 生成的匹配结果进行人工抽检和校准。
2. 设计简历分段解析流程（教育背景/技能/项目经历/实习经历分区提取），结合 DeepSeek 的 JSON 模式输出确保结构化解析；建立常见简历模板的解析规则库，特殊格式兜底人工复核。
3. 实现 API 调用降级策略，当 DeepSeek API 不可用时自动切换至本地缓存的标准化岗位匹配逻辑；对高频咨询问题建立预计算缓存，减少实时 API 依赖；核心业务场景预置静态兜底内容（如常见岗位的技能清单）。
4. 项目启动前 2 天集中进行技术预研与培训，明确技术选型与架构方案；采用成熟稳定的技术栈（如 Spring Boot + Vue），降低学习成本；每日站会同步进度，及时解决技术障碍。

## 管理风险

1. **工期极度紧张风险**：总计 90 人天工作量压缩在 30 个自然日内完成，且功能模块数量多（16 个），涉及学生端、企业端、管理端三条业务线并行开发，进度压力极大。

2. **多角色兼任效率风险**：每位成员均需兼任 2-3 个角色（如组长同时担任项目经理、DBA、主力程序员），且学生端、企业端、管理端三线并行开发，角色频繁切换可能导致注意力分散、工作效率下降。

3. **单人故障点风险**：团队仅 3 人，任一成员因病或意外缺席将导致其负责模块停滞，且每位成员负责的功能模块跨多条业务线，缺乏替补人员。

解决：

1. 采用 MVP 策略，优先交付核心闭环功能（用户画像→能力测评→方向探索→技能学习→岗位匹配），企业端功能和管理后台作为第二阶段优先；每日制定明确的可交付目标，当日事当日毕；预留最后 3 天作为缓冲和集中测试修复期。
2. 以半天为单位划分工作时段（上午专注开发、下午处理管理与测试事务），减少角色切换频率；使用 TAPD/飞书等协作工具统一任务管理，避免信息分散；学生端、企业端、管理端开发按周分阶段推进，避免三线同时并行。
3. 代码每日提交至 Git 仓库，确保知识资产不集中在单机；核心模块（AI 职业方向探索、岗位匹配推荐）采用结对编程，互相了解对方代码；关键文档（数据库设计、API 接口文档、技能词典结构）团队共享，确保任一成员可接手他人工作。

## 其它风险

1. **需求变更风险**：实训项目中指导老师或评审方可能在开发过程中提出需求调整，30 天工期下需求变更代价极高，尤其是功能模块数量较多，任一模块的变更可能引发连锁影响。

2. **岗位数据时效性风险**：本项目严重依赖互联网行业岗位技能标准化词典和供需数据，若数据滞后或不准确，将直接影响 AI 推荐和差距分析的可靠性。

3. **部署与环境风险**：开发环境、测试环境与最终演示环境可能存在差异，导致部署失败或功能异常；DeepSeek API 调用在演示环境需确保网络可达。

解决：

1. 项目启动前完成需求评审与确认，冻结核心需求范围；非核心需求变更记录至"后续迭代"清单，不在本次 30 天内实现；确需变更的核心需求须经全组评估工作量后置换同等规模的低优先级功能。
2. 技能词典和岗位数据采用静态配置文件管理，开发阶段使用预置数据集，演示阶段可通过后台手动更新；关键数据变更记录版本号，确保可追溯。
3. 统一使用 Docker 容器化部署，确保环境一致性；每日进行至少一次集成部署验证；最终演示前 3 天完成完整部署与全流程回归测试；提前与指导老师确认演示环境网络状况，确保 DeepSeek API 可达。
